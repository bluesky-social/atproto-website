## 레이블이란

atproto의 모더레이션은 다음을 포함한 여러 겹의 시스템으로 구성됩니다:

1. API에서 콘텐츠를 필터링하는 네트워크 차단
2. 모더레이션 서비스가 콘텐츠에 적용하는 레이블
3. 뮤트, 차단 등 사용자 컨트롤

클라이언트 애플리케이션을 개발하는 개발자는 레이블과 사용자 컨트롤을 적용하는 방법을 이해해야 합니다.

**레이블**은 AT 생태계의 모든 계정이나 콘텐츠에 대한 메타데이터의 한 형태입니다. 레이블은 *모더레이션 서비스*에 의해 게시되며, 모더레이션 서비스는 애플리케이션에 하드코딩되거나 사용자가 선택합니다. 레이블은 응답의 `labels` 키 하위에 레코드에 첨부됩니다.

레이블은 다음 정보와 함께 게시됩니다:

```markdown
{
  /** DID of the actor who created this label. */
  src: string
  /** AT URI of the record, repository (account), or other resource that this label applies to. */
  uri: string
  /** Optionally, CID specifying the specific version of 'uri' resource this label applies to. */
  cid?: string
  /** The short string name of the value or type of this label. */
  val: string
  /** If true, this is a negation label, overwriting a previous label. */
  neg?: boolean
  /** Timestamp when this label was created. */
  cts: string
}

```

## **레이블 값**

레이블의 *값*이 동작 방식을 결정합니다. 예시 레이블 값으로는 `porn`, `gore`, `spam` 등이 있습니다.

레이블 값은 문자열입니다. 현재는 소문자 a-z 또는 대시 문자만 사용할 수 있습니다(`^[a-z-]+$`). 일부는 `!`로 시작하지만, 이는 전역 레이블 값에만 사용할 수 있습니다.

레이블 값은 해당 정의에 따라 해석됩니다. 정의에는 다음 속성들이 포함됩니다:

- `blurs`: `content`, `media`, `none` 중 하나
- `severity`: `alert`, `inform`, `none` 중 하나
- `defaultSetting`: `hide`, `warn`, `ignore` 중 하나
- `adultOnly`: 불리언(boolean)

다른 정의 속성들도 있지만, 전역 레이블 값에서만 사용됩니다.

## **전역 레이블 값**

프로토콜에 의해 정의된 몇 가지 레이블 값이 있습니다:

- `!hide`: 클릭해서 넘길 수 없는 일반 경고를 콘텐츠에 표시하고, 목록에서 해당 콘텐츠를 필터링합니다. 사용자가 설정을 변경할 수 없습니다.
- `!warn`: 콘텐츠에 일반 경고를 표시하지만 클릭해서 넘길 수 있습니다. 사용자가 설정을 변경할 수 없습니다.
- `!no-unauthenticated`: 해당 레이블을 적용하는 애플리케이션에서 비로그인 사용자가 콘텐츠에 접근할 수 없게 합니다.
- `porn`: 이미지에 경고를 표시하며, 사용자가 18세 이상이고 성인 콘텐츠를 활성화한 경우에만 클릭해서 넘길 수 있습니다.
- `sexual`: `porn`과 동일하게 작동하지만 덜 강렬한 성적 콘텐츠를 처리하기 위한 것입니다.
- `graphic-media`: `porn`과 동일하게 작동하지만 폭력/고어 콘텐츠에 사용됩니다.
- `nudity`: 이미지에 경고를 표시하지만 18세 이상 제한이 없으며 기본값은 무시(ignore)입니다.

전역 레이블 값이 존재하는 데는 두 가지 이유가 있습니다.

첫 번째는 전역으로 정의된 레이블 값만 자기 레이블(즉, 레이블러가 아닌 사용자가 직접 설정)로 사용할 수 있기 때문입니다. porn, sexual, gore, nudity, !no-unauthenticated 레이블이 이러한 이유로 전역입니다.

두 번째는 "사용자 설정 불가" 및 "비로그인 사용자에게만 적용" 등의 특수 동작은 커스텀 레이블에 적용할 수 없기 때문입니다. !hide, !warn, !no-unauthenticated 레이블이 이러한 이유로 전역입니다.

## **커스텀 레이블 값**

레이블러는 자체 레이블 값을 정의할 수 있습니다. 모든 레이블러는 자신이 정의하는 레이블 값의 고유한 네임스페이스를 가집니다. 커스텀 정의는 `!`로 시작하는 것을 제외한 모든 전역 정의를 해당 레이블러에 대해 재정의할 수 있습니다. `!`로 시작하는 것들은 예약되어 있습니다.

두 개의 동작 속성(`blurs`와 `severity`)이 각각 세 가지 값을 가지므로, 커스텀 레이블 값에 대해 9가지 가능한 동작이 있습니다.

| **Blurs** | **Severity** | **설명** |
| --- | --- | --- |
| `content` | `alert` | 콘텐츠를 숨기고, 조회 시 "위험" 경고 레이블을 표시합니다 |
| `content` | `inform` | 콘텐츠를 숨기고, 조회 시 "중립" 정보 레이블을 표시합니다 |
| `content` | `none` | 콘텐츠를 숨깁니다 |
| `media` | `alert` | 콘텐츠의 이미지를 숨기고, 조회 시 "위험" 경고 레이블을 표시합니다 |
| `media` | `inform` | 콘텐츠의 이미지를 숨기고, 조회 시 "중립" 정보 레이블을 표시합니다 |
| `media` | `none` | 콘텐츠의 이미지를 숨깁니다 |
| `none` | `alert` | 콘텐츠에 "위험" 경고 레이블을 표시합니다 |
| `none` | `inform` | 콘텐츠에 "중립" 정보 레이블을 표시합니다 |
| `none` | `none` | 시각적 효과 없음 |

레이블에 사용할 수 있는 정의 예시:

- 괴롭힘: `blurs=content` + `severity=alert`
- 거미 경고: `blurs=media` + `severity=alert`
- 허위 정보: `blurs=none` + `severity=alert`
- 인증된 사용자: `blurs=none` + `severity=inform`
- 큐레이션 다운-레귤레이트: `blurs=none` + `severity=none`

`defaultSetting`은 사용자가 레이블러를 처음 구독할 때 해당 레이블이 어떻게 설정될지를 결정합니다.

`adultOnly`는 성인 콘텐츠가 비활성화된 경우 해당 레이블의 설정을 변경할 수 있는지 여부를 결정합니다.

## **레이블 설정**

사용자는 레이블러의 각 레이블에 대해 숨기기(hide), 경고(warn), 무시(ignore) 중 하나를 선택할 수 있습니다. 숨기기와 경고는 기본적으로 유사하지만, 숨기기는 피드와 목록에서 레이블된 콘텐츠도 필터링합니다. 무시는 레이블을 무시합니다. 설정에서 성인 콘텐츠가 활성화되지 않은 경우, 재정의 없이 강제로 숨기기가 적용됩니다.
