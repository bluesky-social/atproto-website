## 비밀번호 인증

비밀번호 인증을 사용하는 경우, 메인 계정 비밀번호 대신 계정의 [앱 비밀번호](/specs/xrpc#app-passwords)를 생성해서 사용해야 합니다. Bluesky를 사용하는 경우 [계정 설정](https://bsky.app/settings/app-passwords)에서 앱 비밀번호를 생성할 수 있습니다.

    먼저 `PasswordSession`을 가져온 후, `PasswordSession.login()`에 인증 정보를 제공하여 `session`을 생성합니다:

    ```tsx
    import { PasswordSession } from '@atproto/lex-password-session'

    const session = await PasswordSession.login({
      service: 'https://bsky.social', // or your PDS host
      identifier: 'your-handle.bsky.social',
      password: 'your-app-password',
    })
    ```

    최종 사용자 로그인 흐름이 있는 애플리케이션은 앱 비밀번호 세션 대신 [OAuth](/guides/oauth-cli-tutorial) 인증을 사용해야 합니다. 비밀번호 인증은 봇이나 커맨드 라인 도구에 적합합니다.

    두 방법 모두 동일한 인증된 `session`을 생성합니다.
    </TabPanel>
    <TabPanel value="go">
    `atclient` 라이브러리를 가져와서 `LoginWithPasswordHost()`를 호출합니다:

    ```go
    import (
      "context"
      "github.com/bluesky-social/indigo/atproto/atclient"
    )

    func main() {
      ctx := context.Background()

      client, _ := atclient.LoginWithPasswordHost(ctx,
        "https://bsky.social",       // or your PDS host
        "your-handle.bsky.social",   // identifier
        "your-app-password",         // password
        "",                          // authToken (optional 2FA token)
        nil,                         // refresh callback (optional)
      )
    }
    ```

    최종 사용자 로그인 흐름이 있는 애플리케이션은 앱 비밀번호 세션 대신 [OAuth](/guides/go-oauth-cli-tutorial) 인증을 사용해야 합니다. 비밀번호 인증은 봇이나 커맨드 라인 도구에 적합합니다.
    </TabPanel>
  </TabPanels>
</TabGroup>

## API 토큰

로그인에 대한 속도 제한은 일반적으로 다른 API 작업보다 낮습니다. 짧은 시간 내에 로그인 메서드를 여러 번 호출하면 속도 제한이 발생할 수 있습니다. 인증을 위해 세션당 로그인 메서드를 한 번만 호출하면 됩니다.

로그인 메서드가 호출하는 `com.atproto.server.createSession` API 엔드포인트는 두 개의 API 토큰이 포함된 세션 객체를 반환합니다:
* `accessJwt`: 요청 인증에 사용되지만 몇 분 후 만료되는 액세스 토큰
* `refreshJwt`: 더 오래 지속되며 새로운 액세스 토큰으로 세션을 갱신하는 데만 사용되는 갱신 토큰
