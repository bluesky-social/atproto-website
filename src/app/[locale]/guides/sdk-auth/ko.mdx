## API 토큰

로그인 속도 제한은 일반적으로 다른 API 작업보다 낮습니다. 짧은 시간 내에 로그인 메서드를 여러 번 호출하면 속도 제한이 발생할 수 있습니다. 인증하려면 세션당 로그인 메서드를 한 번만 호출하면 됩니다.

로그인 메서드가 호출하는 `com.atproto.server.createSession` API 엔드포인트는 두 개의 API 토큰이 포함된 세션 객체를 반환합니다:
* `accessJwt`: 요청 인증에 사용되지만 몇 분 후에 만료되는 액세스 토큰
* `refreshJwt`: 더 오래 지속되며 새 액세스 토큰으로 세션을 업데이트하는 데만 사용되는 갱신 토큰
