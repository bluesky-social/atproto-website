[Permission sets](https://atproto.com/specs/lexicon#permission-set) are a way to bundle similar OAuth scopes together into a single group to make them easier for people to understand when authorizing your app. For example, if you wanted to build an app that can post to, but not update the permissions of, your Bluesky account you might request `include:app.bsky.authCreatePosts?aud=did:web:api.bsky.app`. Permission sets let you request access to someone's repository as well as the XRPC endpoints their app might define.

If your application defines custom Lexicons or endpoints, it's a best practice to also define a permission set instead of requesting the much broader `transition:generic` scope. This tool outputs a Lexicon that you can publish with a tool like goat via `goat lex publish`.
