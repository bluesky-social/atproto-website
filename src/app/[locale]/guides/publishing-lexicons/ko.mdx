## goat 사용하기

Lexicon은 기계가 읽을 수 있고 네트워크에서 접근 가능하도록 설계되었습니다. 현재 Lexicon이 네트워크에서 반드시 사용 가능해야 하는 것은 아니지만, 메서드 소비자가 단일 표준적이고 권위 있는 표현에 접근할 수 있도록 Lexicon을 배포하는 것을 강력히 권장합니다.

Lexicon 생성 및 배포에는 커맨드라인 도구인 [`goat`](https://github.com/bluesky-social/goat)를 사용할 수 있습니다. Lexicon 관련 기능은 [`goat lex`](https://github.com/bluesky-social/goat?tab=readme-ov-file#lexicon-development) 아래에서 사용할 수 있습니다:

- AT 네트워크에 스키마 배포 및 동기화
- 스키마 비교(diff), 린팅, 스키마 진화 규칙 검증

`goat`는 macOS와 Linux에서 [Homebrew](https://formulae.brew.sh/formula/goat)를 통해 설치할 수 있습니다:

```shell
brew install goat
```

프로젝트 디렉터리에서 기존 Lexicon을 다운로드할 수 있으며, `./lexicons/`에 JSON 파일로 저장됩니다:

```markdown
$ goat lex pull com.atproto.repo.strongRef com.atproto.moderation. app.bsky.actor.profile
 🟢 com.atproto.repo.strongRef
 🟢 com.atproto.moderation.defs
 🟢 com.atproto.moderation.createReport
 🟢 app.bsky.actor.profile
```

`goat lex new record`로 새 Lexicon record를 생성할 수도 있습니다:

```markdown
$ goat lex new record dev.project.thing

$ open ./lexicons/dev/project/thing.json
```

그리고 `goat lex publish`로 배포할 수 있습니다:

```markdown
$ goat lex publish
 🟢 dev.project.thing
```

<Note>
Lexicon은 다른 record와 마찬가지로 *본인의 repository*에 배포됩니다. 즉, 배포할 때 `goat`로 로그인해야 합니다! `goat account login`을 사용하거나 `GOAT_USERNAME`과 `GOAT_PASSWORD` 환경 변수를 설정하세요.
</Note>

새 Lexicon 생성에 대한 안내는 [Lexicon 스타일 가이드](/guides/lexicon-style-guide)를 참고하세요.
