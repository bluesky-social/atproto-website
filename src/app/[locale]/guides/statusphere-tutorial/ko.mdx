## 필수 조건
본 튜토리얼은 [NextJS를 활용한 OAuth 튜토리얼](/guides/oauth-tutorial)에서 생성한 앱을 기반으로 합니다. 시작하기 전에 NextJS OAuth 앱을 완성해 두셔야 합니다. 해당 단계를 건너뛰고 싶다면 [Cookbook](https://github.com/bluesky-social/nextjs-oauth-tutorial/tree/129cf28b4146ae709e0dac43ce20154ab5237559)에서 완성된 OAuth 앱을 복제할 수 있습니다.
다음이 설치되어 있어야 합니다:
- Node.js 18 이상
- Go 1.25 이상
- pnpm 패키지 관리자
[homebrew](https://brew.sh/)가 지원하는 플랫폼에서는 다음 명령어로 설치할 수 있습니다:
```bash
brew install node pnpm go
```
기존 프로젝트에 다음 Atproto 라이브러리를 추가하세요:
```bash
pnpm add @atproto/common-web @atproto/lex @atproto/syntax @atproto/tap
```
또한 `lex` CLI 도구를 전역으로 설치해야 합니다:
```bash
npm install -g @atproto/lex
```
프로젝트에 어휘집(Lexicon)을 추가하는 것으로 시작합니다.
## 파트 1: 어휘집
[어휘집](/guides/lexicon)은 Atproto 내 레코드의 스키마를 정의합니다. 본 튜토리얼을 위해 저희는 이미 새로운 "Statusphere" Lexicon을 `xyz.statusphere.status`에서 확인할 수 있습니다. `lex`를 사용하여 이 Lexicon을 다운로드하고 프로젝트에 빌드할 수 있습니다:
```bash
lex install xyz.statusphere.status
```
원한다면 다운로드된 Lexicon 파일을 `lib/lexicons/ xyz.statusphere.status.json`에서 확인할 수 있습니다. 여기서 `lex build`를 실행하면 설치된 모든 레식콘에 대한 TypeScript 타입을 생성할 수 있습니다:
```bash
lex build --importExt=\"\"
```
<Note>
필요하다면 이 명령어를 `package.json`에 추가할 수 있지만, 사전이 빌드된 상태에서 이 튜토리얼에서는 더 이상 사용하지 않을 것입니다. `lex install`로 프로젝트에 다른 사전을 추가할 경우, ` --override` 플래그를 사용하여 `lex build`를 다시 실행해야 합니다.
</Note>
지금 해야 할 한 가지 더는 `lib/auth/client.ts` 파일의 `SCOPE` 상수를 업데이트하여 이 컬렉션에 대한 액세스를 요청하는 것입니다. Atproto 권한 범위는 Lexicons이므로 앱이 요청하는 범위 목록에 `xyz.statusphere.status`를 추가할 수 있습니다:
```typescript
export const SCOPE = "atproto repo:xyz.statusphere.status";
```
이제 데이터베이스 스키마 생성을 진행할 수 있습니다.

## 파트 3: 상태 제출
먼저 사용자가 자신의 [PDS](/guides/the-at-stack#pds)에 상태를 작성할 수 있는 기능을 구축합니다. `app/api/status/route.ts`에 새 API 경로를 생성하세요:
```typescript
import { NextRequest, NextResponse } from "next/server";
import { Client } from "@atproto/lex";
import { getSession } from "@/lib/auth/session";
import { getOAuthClient } from "@/lib/auth/client";
import * as xyz from "@/src/lexicons/xyz";
export async function POST(request: NextRequest) {
  const session = await getSession();

if (!session) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }
  const { status } = await request.json();
  if (!status || typeof status !== 'string') {
    return NextResponse.json({ error: "Status is required" }, { status: 400 });

}
  const client = await getOAuthClient();
  const oauthSession = await client.restore(session.did);
  const lexClient = new Client(oauthSession);
  const createdAt = new Date().toISOString();
  const res = await lexClient.create(xyz.statusphere.status, {
    status,

createdAt,
  });
  return NextResponse.json({
    success: true,
    uri: res.uri,
  });
```
이 코드는 사용자가 로그인했는지 확인하고, OAuth 세션을 사용해 lex `Client`를 생성한 후 제출된 상태 텍스트로 새로운 `xyz.statusphere.status` 레코드를 생성합니다.

## 파트 4: Tap을 이용한 실시간 동기화
[Tap](/guides/backfilling#using-tap)은 AT Protocol 레코드를 동기화하고 스트리밍하는 가장 좋은 방법입니다. 데이터베이스의 새 레코드를 동기화하기 위해 Tap [TypeScript 라이브러리](https://www.npmjs.com/package/@atproto/tap)를 사용할 것입니다. `lib/tap/index.ts`를 생성하세요:
```typescript
import { Tap } from "@atproto/tap";
const TAP_URL = process.env.TAP_URL || "http://localhost:2480";
let _tap: Tap | null = null;
export const getTap = (): Tap => {
  if (!_tap) {
    _tap = new Tap(TAP_URL);

}
  return _tap;
};
```
다음으로, Tap 이벤트 처리를 위한 데이터베이스 쿼리를 포함하는 `lib/db/queries.ts`를 생성합니다:
```typescript
import { getDb, AccountTable, StatusTable, DatabaseSchema } from ".";
import { AtUri } from "@atproto/syntax";
import { Transaction } from "kysely";
export async function getAccountStatus(did: string) {
  const db = getDb();
  const status = await db
    .selectFrom("status")
    .selectAll()
    .where("authorDid", "=", did)
    .orderBy('createdAt', "desc")

.limit(1)
    .executeTakeFirst();
  return status ?? null;
}
export async function insertStatus(data: StatusTable) {
  getDb()
    .transaction()
    .execute(async (tx) => {
      await tx
        .insertInto("status")
        .values(data)

.onConflict((oc) =>
          oc.column("uri").doUpdateSet({
            status: data.status,
            createdAt: data.createdAt,
            indexedAt: data.indexedAt,
          }),
        )
        .execute();
      setCurrStatus(tx, data.authorDid);

});
}
export async function deleteStatus(uri: AtUri) {
  await getDb()
    .transaction()
    .execute(async (tx) => {
      await tx.deleteFrom("status").where('uri', "=", uri.toString()).execute();
      await setCurrStatus(tx, uri.hostname);

});
}
export async function upsertAccount(data: AccountTable) {
  await getDb()
    .insertInto("account")
    .values(data)
    .onConflict((oc) =>
      oc.column("did").doUpdateSet({
        handle: data.handle,
        active: data.active,
      }),
    )

.execute();
}
export async function deleteAccount(did: string) {
  await getDb().deleteFrom("account").where("did", "=", did).execute();
  await getDb().deleteFrom("status").where('authorDid', "=", did).execute();
}
// 사용자의 "현재" 상태를 업데이트하는 헬퍼 함수 (트랜잭션 내부에서 사용)
async function setCurrStatus(tx: Transaction<DatabaseSchema>, did: string) {
  // 모든 사용자의 상태에 대한 현재 플래그 지우기
  await tx
    .updateTable("status")
    .set({ current: 0 })
    .where("authorDid", "=", did)
    .where('current', "=", 1)
    .execute ();
  // 가장 최근 상태를 현재 상태로 설정
  await tx
    .updateTable("status")
    .set({ current: 1 })
    .where("uri", "=", (qb) =>
      qb
        .selectFrom("status")
        .select("uri")
        .where('authorDid', "=", did)

.orderBy("createdAt", "desc")
        .limit(1),
    )
    .execute();
}
```
`setCurrStatus` 헬퍼는 사용자당 가장 최근 상태만 `current = 1`이 되도록 보장합니다. 이는 나중에 사용될 것입니다.
Tap은 `app/api/webhook/route.ts`의 웹훅 엔드포인트를 통해 이벤트를 전달합니다:
```typescript
import { NextRequest, NextResponse } from "next/server";
import { parseTapEvent, assureAdminAuth } from "@atproto/tap";
import { AtUri } from "@atproto/syntax";
import {

upsertAccount,
  insertStatus,
  deleteStatus,
  deleteAccount,
} from "@/lib/db/queries";
import * as xyz from "@/src/lexicons/xyz";
const TAP_ADMIN_PASSWORD = process.env.TAP_ADMIN_PASSWORD;
export async function POST(request: NextRequest) {
  // 요청이 TAP 서버에서 온 것인지 확인
  if (TAP_ADMIN_PASSWORD) {
    const authHeader = request.headers.get("Authorization");
    if (!authHeader) {

return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
    }
    try {
      assureAdminAuth(TAP_ADMIN_PASSWORD, authHeader);
    } catch {
      return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
    }

}
  const body = await request.json();
  const evt = parseTapEvent(body);
  // 계정/신원 변경 처리
  if (evt.type === "identity") {
    if (evt.status === "deleted") {

await deleteAccount(evt.did);
    } else {
      await upsertAccount({
        did: evt.did,
        handle: evt.handle,
        active: evt.isActive ? 1 : 0,
      });
    }
  }
  // 상태 레코드 변경 처리
  if (evt.type === "record") {

const uri = AtUri.make(evt.did, evt.collection, evt.rkey);
    if (evt.action === "create" || evt.action === "update") {
      let record: xyz.statusphere.status.Main;
      try {
        record = xyz.statusphere.status.$parse(evt.record);

} catch {
        return NextResponse.json({ success: false });
      }
      await insertStatus({
        uri: uri.toString(),
        authorDid: evt.did,
        status: record.status,

createdAt: record.createdAt,
        indexedAt: new Date().toISOString(),
        current: 1,
      });
    } else if (evt.action === "delete") {
      await deleteStatus(uri);
    }
  }
  return NextResponse.json( { success: true });
}
```
Tap은 네트워크 어디에서든 레코드가 변경될 때 웹훅 이벤트를 전송합니다. 요청은 공유 비밀 키를 사용하여 검증됩니다. `identity` 이벤트의 경우 계정 캐시를 업데이트하고, `record` 이벤트의 경우 레코드가 어휘집(Lexicon)과 일치하는지 확인한 후 데이터베이스에 삽입/업데이트/삭제합니다.
현재 사용자의 상태를 가져와 전달하도록 `app/page.tsx`를 다시 업데이트하세요:

## 파트 5: 핸들 및 피드 표시
마지막으로 사용자 핸들과 모든 사용자의 상태 피드를 표시하는 기능을 추가합니다.
먼저 `lib/db/queries.ts`에 추가 임포트를 추가하세요:
```typescript
import { getHandle } from "@atproto/common-web";

```
그런 다음 계정 핸들과 상태를 가져오는 추가 함수를 추가합니다:
```typescript
export async function getAccountHandle(did: string): Promise<string | null> {
  const db = getDb();
  // Tap을 통해 계정을 추적하고 계정 정보를 얻은 경우, 해당 정보에서 로드합니다
  const account = await db
    .selectFrom("account")
    .select("handle")

.where("did", "=", did)
    .executeTakeFirst();
  if (account) return account.handle;
  // 그렇지 않으면 Tap을 통해 계정 DID를 해결합니다. Tap은 신원 캐싱을 제공합니다.
  try {
    const didDoc = await getTap().resolveDid(did);
    if (!didDoc) return null;
    return getHandle(didDoc) ?? null;

} catch {
    return null;
  }
}
export async function getRecentStatuses(limit = 5) {
  const db = getDb ();
  return db
    .selectFrom("status")
    .innerJoin("account", "status.authorDid", "account.did")
    .selectAll()
    .orderBy('createdAt', "desc")
    .limit(limit)
    .execute();
}
export async function getTopStatuses(limit = 10) {
  const db = getDb ();
  return db
    .selectFrom("status")
    .select(["status", db.fn.count("uri").as("count")])
    .where("current", "=", 1)
    .groupBy('status')
    .orderBy("count", "desc")
    .limit(limit)
    .execute();
}
```

```tsx
function timeAgo(dateString: string): string {
  const now = Date.now();
  const then = new Date(dateString).getTime();
  const seconds = Math.floor((now - then) / 1000);
  if (seconds < 60) return "방금 전";

const minutes = Math.floor(seconds / 60);
  if (minutes < 60) return `${minutes}m`;
  const hours = Math.floor(minutes / 60);
  if (hours < 24) return `${hours}h`;
  const days = Math.floor (hours / 24);
  return `${days}d`;
}
```
