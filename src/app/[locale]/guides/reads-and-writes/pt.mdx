## Registros AT
O Protocolo AT distribui os dados do usuário por diferentes nós da sua rede, utilizando os blocos de construção essenciais da web. O Protocolo AT interconecta aplicativos para que seus back - ends compartilhem o estado, incluindo contas de usuário e conteúdo, em repositórios de dados individuais. Muitas operações que você realizará ao trabalhar com aplicativos Atproto envolvem a leitura e a gravação desses repositórios de dados.

Para obter mais contexto, consulte [Atproto para engenheiros de sistemas distribuídos](/articles/atproto-for-distsys-engineers) e [O Ethos Atproto](/articles/atproto-ethos).
## Pré - requisitos
A leitura e a gravação de repositórios de dados AT requerem um cliente autenticado. Você pode se autenticar com [autenticação por senha](/guides/sdk-auth) ou [OAuth](/guides/oauth-patterns). Os guias nesta seção presumem que você já configurou um cliente autenticado.

        Para fazer solicitações de API, você precisará especificar um [Lexicon](/guides/lexicon)  específico. Por exemplo, as publicações do Bluesky estão disponíveis no léxico `app.bsky.feed.post`.

          Use `lex` para instalar os léxicos `app.bsky.feed.post` e `app.bsky.actor.profile`, conforme descrito em [Instalando léxicos](/guides/installing-lexicons):
```bash
npm install -g @atproto/lex
    
lex install app.bsky.feed.post app.bsky.actor.profile
lex build
```
Isso irá gerar tipos TypeScript para os Lexicons em seu projeto. Você pode usar esta instrução `import` para acessar todos os Lexicons instalados:
```tsx
import * as app from './lexicons/app.js'
```
    
Isso lhe dará acesso aos métodos da API para seus Lexicons. Agora você está pronto para começar a [fazer solicitações](/guides/reading-data)!
