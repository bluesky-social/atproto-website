## AT 레코드
AT 프로토콜은 웹의 핵심 구성 요소를 활용하여 사용자 데이터를 네트워크의 다양한 노드에 분산합니다.AT 프로토콜은 애플리케이션을 상호 연결하여 백엔드가 개별 데이터 저장소에서 사용자 계정 및 콘텐츠를 포함한 상태를 공유하도록 합니다.Atproto 애플리케이션 작업 시 수행하는 많은 작업은 이러한 데이터 저장소의 읽기 및 쓰기와 관련됩니다.

더 자세한 내용은 [분산 시스템 엔지니어를 위한 Atproto](/articles/atproto-for-distsys-engineers) 및 [Atproto의 정신](/articles/atproto-ethos) 을 참조하세요.
## 필수 조건
AT 데이터 저장소의 읽기 및 쓰기에는 인증된 클라이언트가 필요합니다. [비밀번호 인증](/guides/sdk-auth) 또는 [OAuth](/guides/oauth-patterns) 로 인증할 수 있습니다. 이 섹션의 가이드는 이미 인증된 클라이언트를 설정했다고 가정합니다.

        API 요청을 수행하려면 특정 [Lexicon](/guides/lexicon) 을 지정해야 합니다.예를 들어, Bluesky 게시물은 `app.bsky.feed.post` 어휘에서 사용할 수 있습니다.

    [어휘 설치](/guides/installing-lexicons) 에 설명된 대로 `lex`를 사용하여 `app.bsky.feed.post` 및 `app.bsky.actor.profile` 어휘를 설치하세요:
```bash
npm install -g @atproto/lex
    
lex install app.bsky.feed.post app.bsky.actor.profile
lex build
```
    이렇게 하면 프로젝트에 Lexicons에 대한 TypeScript 타입이 생성됩니다.설치된 모든 Lexicons에 접근하려면 이 `import` 문을 사용하세요:
```tsx
import * as app from './lexicons/app.js'
```
    
이를 통해 Lexicons의 API 메서드에 접근할 수 있습니다. 이제 [요청을 시작](/guides/reading-data) 할 준비가 되었습니다!
  </TabPanel>
