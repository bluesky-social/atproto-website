## 레코드 목록 조회

<TabGroup defaultValue="ts">
  <TabList>
    <Tab value="ts">TypeScript</Tab>
    <Tab value="go">Go</Tab>
  </TabList>
  <TabPanels>
    <TabPanel value="ts">
    `client.list()`를 사용하여 사용자의 데이터 저장소에서 레코드를 읽을 수 있습니다:

    ```tsx
    const posts = await client.list(app.bsky.feed.post, {
      limit: 10,
      repo: 'alex.bsky.team',
    })
    ```

    각 Lexicon에는 고유한 매개변수가 있습니다. `app.bsky.feed.post`의 경우 `repo`(사용자의 핸들 또는 [DID](#did-resolution))와 반환할 게시물 수에 대한 `limit`을 지정할 수 있습니다.
    </TabPanel>
    <TabPanel value="go">
    `RepoListRecords()`를 사용하여 사용자의 데이터 저장소에서 게시물을 읽을 수 있습니다:

    ```go
    resp, _ := comatproto.RepoListRecords(ctx, client,
        "app.bsky.feed.post",  // lexicon
        "",                    // cursor
        10,                    // limit
        "alex.bsky.team",      // handle or DID
        false,                 // reverse
    )

    for _, record := range resp.Records {
        fmt.Printf("Record: %s\n", record.Uri)
    }
    ```
    </TabPanel>
  </TabPanels>
</TabGroup>

이 Lexicon의 전체 내용은 [`app.bsky.feed.post`](https://lexicon.garden/lexicon/did:plc:4v4y5r3lwsbtmsxhile2ljac/app.bsky.feed.post)를 참조하세요.

    ```go
        "",                    // cid (empty for latest)
        "app.bsky.feed.post",  // lexicon
        "alex.bsky.team",      // handle or DID
        "3mbl5xdw4yk2r",       // rkey
    )

    fmt.Printf("URI: %s\n", resp.Uri)
    // resp.Value contains the record data
    ```

## DID 해석

atproto에서 사용자 데이터는 [DID](/guides/sync#data-repositories)([분산 식별자](https://en.wikipedia.org/wiki/Decentralized_identifier)라고도 함)를 통해 해석됩니다. 대부분의 AT Protocol 애플리케이션은 사용자 계정을 등록할 때 [`did:plc` DID](/guides/the-at-stack#did-plc)를 사용합니다.

`com.atproto.identity.resolveHandle` 엔드포인트를 사용하여 알려진 사용자 이름을 DID로 해석할 수 있습니다. 즉, `handle=` 인수와 함께 브라우저에서 이 주소를 방문하여 DID를 조회할 수 있습니다:

```
https://bsky.social/xrpc/com.atproto.identity.resolveHandle?handle=youraccount.bsky.social
```

```
{
did: "did:plc:3vekukmab2djjeengw2o3wln"
}
```

TypeScript SDK는 현재 인증된 사용자의 DID를 `client.did`로도 제공합니다:

```tsx
const your_did = client.did
```

## 비인증 읽기

일부 API 엔드포인트는 인증 없이도 접근할 수 있습니다. 새 `Client`에 공개 API URL을 전달하여 비인증 클라이언트를 생성할 수 있습니다:

<TabGroup defaultValue="ts">
  <TabList>
    <Tab value="ts">TypeScript</Tab>
    <Tab value="go">Go</Tab>
  </TabList>
  <TabPanels>
    <TabPanel value="ts">
    ```tsx
    import { Client } from '@atproto/lex'
    import * as app from './lexicons/app.js'

    const client = new Client('https://public.api.bsky.app')
    ```

    인증된 클라이언트와 마찬가지로, 사용자 프로필과 같은 공개 레코드를 `.get()`할 수 있습니다:

    ```tsx
    const profile = await client.call(app.bsky.actor.getProfile, {
      actor: 'alex.bsky.team',
    })
    ```

    동일한 SDK 클라이언트 패턴을 사용하여 모든 Lexicon에 API 호출을 할 수 있습니다. 더 많은 예시는 [SDK 문서](https://www.npmjs.com/package/@atproto/lex)를 참조하세요.
    </TabPanel>
    <TabPanel value="go">
    ```go
    import (
        "context"
        appbsky "github.com/bluesky-social/indigo/api/bsky"
        "github.com/bluesky-social/indigo/atproto/atclient"
    )

    ctx := context.Background()

    client := atclient.NewAPIClient("https://public.api.bsky.app")
    ```

    인증된 클라이언트와 마찬가지로, 사용자 프로필과 같은 공개 레코드를 `.get()`할 수 있습니다:

    ```go
    profile, _ := appbsky.ActorGetProfile(ctx, client, "alex.bsky.team")

    fmt.Printf("Handle: %s\n", profile.Handle)
    fmt.Printf("Display Name: %s\n", *profile.DisplayName)
    fmt.Printf("Followers: %d\n", *profile.FollowersCount)
    ```

    동일한 SDK 클라이언트 패턴을 사용하여 모든 Lexicon에 API 호출을 할 수 있습니다. 더 많은 예시는 [Go SDK 라이브러리](https://pkg.go.dev/github.com/bluesky-social/indigo)를 참조하세요.
    </TabPanel>
  </TabPanels>
</TabGroup>

다음으로 사용자의 데이터 저장소에 [쓰기](/guides/writing-data)를 시도해볼 수 있습니다.
